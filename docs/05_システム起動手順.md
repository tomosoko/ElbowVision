# 05 システム起動手順

## 通常起動（開発時）

2つのターミナルを開いて、それぞれ実行する。

### ターミナル 1 : FastAPI バックエンド

```bash
cd ~/develop/Dev/vision/ElbowVision/elbow-api
source venv/bin/activate
uvicorn main:app --host 0.0.0.0 --port 8000
```

起動確認:
```
INFO:     Started server process
INFO:     Waiting for application startup.
INFO:     Application startup complete.
INFO:     Uvicorn running on http://0.0.0.0:8000
```

### ターミナル 2 : Next.js フロントエンド

```bash
cd ~/develop/Dev/vision/ElbowVision/elbow-frontend
npm run dev
```

起動確認:
```
▲ Next.js 15.x.x
   - Local:   http://localhost:3000
   - Ready in Xs
```

### 起動後にブラウザで開く

```
http://localhost:3000
```

## API エンドポイント一覧

| エンドポイント | メソッド | 説明 |
|-------------|---------|------|
| `/api/health` | GET | 疎通確認・エンジン状態 |
| `/api/upload` | POST | 画像アップロード・base64変換 |
| `/api/analyze` | POST | 肘関節解析（ランドマーク + 角度 + エッジ検証 + ポジショニング補正） |
| `/api/gradcam` | POST | Grad-CAM XAI ヒートマップ生成（ConvNeXt訓練後に有効） |

### /api/analyze レスポンス構造

```json
{
  "success": true,
  "landmarks": {
    "angles": {
      "carrying_angle": 10.5,     // AP像のみ（LAT像はnull）
      "flexion": null,             // LAT像のみ（AP像はnull）
      "pronation_sup": 2.1,        // 回内(+)/回外(-)
      "varus_valgus": 1.3          // 外反(+)/内反(-)
    },
    "qa": {
      "view_type": "AP",           // AP / LAT
      "score": 95,
      "status": "GOOD",            // GOOD / FAIR / POOR
      "positioning_advice": "..."  // ポジショニングコメント
    }
  },
  "edge_validation": {
    "edge_angle": 11.2,            // エッジから算出した角度
    "agreement_deg": 0.7,          // プライマリ角度との差
    "confidence": "high",          // high / medium / low
    "note": "エッジ一致 ✓ ..."
  },
  "positioning_correction": {
    "view_type": "AP",
    "epic_separation_px": 80.0,    // 外顆間距離（体格指標）
    "epic_ratio": 0.113,           // 外顆間距離 / 画像対角線
    "rotation_error": 3.5,         // 理想位からの回旋ズレ量（°）
    "rotation_level": "good",      // good / minor / major
    "rotation_advice": "回外位が適切です...",
    "flexion_deg": null,           // LAT像のみ（AP像はnull）

    "carrying_angle": 10.8,
    "flexion": null,
    "model": "ConvNeXt-Small"
  }
}
```

### cURL でのテスト例

```bash
# ヘルスチェック
curl http://localhost:8000/api/health

# 画像解析
curl -X POST http://localhost:8000/api/analyze \
  -F "file=@/path/to/elbow.png"

# Grad-CAM（外反角ターゲット）
curl -X POST "http://localhost:8000/api/gradcam?target=carrying" \
  -F "file=@/path/to/elbow.png"
```

## Docker での起動（本番・デモ用）

```bash
cd ~/develop/Dev/vision/ElbowVision
docker-compose up --build
```

停止:
```bash
docker-compose down
```

> Docker を使うとポート設定や環境構築なしで一発起動できる。
> デモや発表時に便利。

## よくある起動エラー

### APIが起動しない

```
ModuleNotFoundError: No module named 'fastapi'
```
→ 仮想環境が有効化されていない。`source venv/bin/activate` を実行。

```
Address already in use: 8000
```
→ すでにAPIが起動中。`kill $(lsof -ti:8000)` で停止してから再起動。

### フロントエンドがAPIに繋がらない

- API（8000番）が起動しているか確認: `curl http://localhost:8000/api/health`
- ブラウザのコンソールで CORS エラーが出ていないか確認
- `CORS_ORIGINS` 環境変数が `http://localhost:3000` になっているか確認

### Next.js のポートが 3000 以外になっている

```bash
# ポートを明示的に指定
PORT=3000 npm run dev
```

---

## プロセス確認・停止

```bash
# 8000番ポートを使っているプロセスを確認
lsof -i :8000

# 3000番ポートを使っているプロセスを確認
lsof -i :3000

# 強制停止
kill -9 $(lsof -ti:8000)
kill -9 $(lsof -ti:3000)
```
