# ElbowVision プレ研究手順書

## 概要

本手順書はファントムを使った最初の動作確認実験（プレ研究）の全工程を記述する。
ゴール：「AIが肘X線のキーポイントを検出できる」ことをファントム画像で実証する。

---

## 環境について

| 作業 | 推奨環境 | 理由 |
|------|---------|------|
| DRR生成・前処理 | **Mac Mini M4 Pro**（ローカル） | 高速CPU + MPS GPU、MacBook Proより大幅に高速 |
| YOLO訓練 | **Mac Mini M4 Pro**（ローカル） | MPS GPU で十分高速、Colab不要 |
| ConvNeXt訓練 | **Mac Mini M4 Pro**（ローカル） | MPS GPU 対応、アップロード手間なし |
| 結果確認・可視化 | Mac Mini M4 Pro（ローカル） | |

> **Note:** Mac Mini M4 Pro の導入により Google Colab は不要になった。
> DRR生成から訓練・検証まですべてローカルで完結する。

---

## 全体フロー

```
STEP 1: 合成ファントムCT生成（M4 Pro ローカル・5分）
    ↓
STEP 2: DRR生成（M4 Pro ローカル・5〜15分）
    ↓
STEP 3: 生成画像の目視確認（ローカル・5分）

    ↓

    ↓
STEP 6: 実ファントム撮影（放射線室・半日）
    ↓
STEP 7: 実X線データで検証（ローカル・数時間）
```

## STEP 3: 生成画像の目視確認

DRR画像を確認する。以下が見えていればOK：
- 上腕骨幹部（縦長の白い骨幹）
- 上顆部（横方向に広がった部分）
- 橈骨・尺骨（前腕の2本の骨）

```bash
# macOS でランダムに数枚開く
open data/yolo_dataset/images/train/elbow_00000.png
open data/yolo_dataset/images/train/elbow_00050.png
open data/yolo_dataset/images/train/elbow_00100.png
```

**NGな場合（よくある問題）:**

| 症状 | 原因 | 対処 |
|------|------|------|
| 真っ黒 / 真っ白 | HUウィンドウ不良 | create_phantom.py のHU値を確認 |
| 骨が見えない | 骨閾値が高すぎ | Otsuログの閾値を確認 |
| 上下が逆 | CT向き検出ミス | --laterality を明示指定 |

## STEP 4: YOLO + ConvNeXt 訓練（M4 Pro ローカル）

Mac Mini M4 Pro の MPS GPU を使い、すべてローカルで訓練する。
Google Colab へのアップロードは不要。

### YOLOv8-Pose 訓練

```bash
cd /Users/kohei/develop/Dev/vision/ElbowVision
source elbow-api/venv/bin/activate

python elbow-train/train_yolo_pose.py
```

**重要設定:**
- `imgsz=256`（target_size=256 に合わせる）
- `fliplr=0.5`（flip_idx [0,2,1,3,4,5] で上顆左右入れ替え）
- `yolov8s-pose.pt`（nano より精度高い small モデル）
- `device='mps'`（M4 Pro の MPS GPU を使用）

### ConvNeXt 訓練

```bash
python elbow-api/training/train_angle_predictor.py \
  --csv  data/yolo_dataset_v2/convnext_labels.csv \
  --imgs data/yolo_dataset_v2/
```

### 訓練中に確認すること

- `box_loss` と `pose_loss` が下がり続けているか
- 途中でNaNが出ていないか
- 約20〜40分で150 epoch完了（M4 Pro MPS GPU 使用時）

> **参考:** Google Colab で訓練する場合は `elbow-train/elbow_colab_train.ipynb` を使用できるが、
> M4 Pro があれば Colab は不要。

## STEP 7: 実X線データで検証

合成DRRで訓練したモデルを実X線に適用し、
ドメインギャップ（合成と実画像の差）を評価する。

```python
# 実X線での推論精度確認（ローカル実行）
results_real = model_best.predict(
    source='data/xray_png/',
    conf=0.3,
    save=True,
)
```

**期待される結果と対策:**

| 結果 | 評価 | 次のアクション |
|------|------|--------------|
| 実X線でも検出できた | 合格 | 本格データ収集へ |
| 検出できるが座標がズレる | 許容範囲 | 実X線でFine-tuning |
| ほとんど検出できない | 要改善 | DRR生成パラメータを実X線に近づける |

## トラブルシューティング

### DRR生成が遅い

```bash
# n_ap, n_lat を減らして動作確認だけする
python elbow-train/elbow_synth.py \
  --ct_dir data/raw_dicom/ct/ \
  --out_dir data/yolo_dataset_test/ \
  --n_ap 10 --n_lat 30
```

### M4 Pro でメモリエラー（MPS）

```python
# batch サイズを下げる
model.train(..., batch=8)
# MPS メモリが不足する場合は batch=4 まで下げる
```

### キーポイントが端に張り付く

`auto_detect_landmarks` の検出失敗。`create_phantom.py` のランドマーク定義がCTに合っているか確認する。
