# 04 モデル訓練手順（YOLOv8-Pose）

## 前提条件

- `data/images/train/` に訓練用PNG（推奨: 64枚以上）
- `data/images/val/` に検証用PNG（推奨: 16枚以上）
- `data/labels/train/` にYOLOキーポイントラベル
- `data/labels/val/` にYOLOキーポイントラベル
- 画像ファイルとラベルファイルのファイル名（拡張子除く）が一致していること

---

## Step 1 : データセットの確認

```bash
cd /Users/kohei/ElbowVision_Dev

echo "=== val ===" && ls data/images/val/*.png | wc -l && ls data/labels/val/*.txt | wc -l
```

両方の数が一致していれば準備完了。

## Step 3 : 訓練の実行

```bash

python elbow-train/train_yolo_pose.py
```

### 訓練中に表示されるログの見方

```
Epoch  1/100  ...  box_loss: 2.341  pose_loss: 3.812  ← 最初は高い
Epoch 20/100  ...  box_loss: 0.823  pose_loss: 1.241  ← 下がってきたら正常
Epoch 50/100  ...  box_loss: 0.412  pose_loss: 0.634
              ↑ これが下がり続けていれば順調
```

### 早期停止（Early Stopping）

`patience=20` の設定により、20エポック改善がなければ自動停止する。
100エポック未満で止まっても問題ない。

### 訓練時間の目安

| 環境 | 目安 |
|-----|------|
| MacBook Pro M3 (MPS) | 80枚 × 100epoch ≈ 20〜40分 |
| MacBook Pro M1/M2 (MPS) | 80枚 × 100epoch ≈ 30〜60分 |
| CPU のみ | 80枚 × 100epoch ≈ 2〜4時間 |
| GPU (CUDA) | 80枚 × 100epoch ≈ 10〜20分 |

### 精度指標の確認

```bash
cat elbow-api/runs/pose/elbowvision_pose_model/results.csv | \
  python3 -c "
import csv, sys
rows = list(csv.DictReader(sys.stdin))
last = rows[-1]
print(f'pose_mAP50:   {float(last.get(\"metrics/pose_mAP50\", 0)):.3f}')
print(f'pose_mAP50-95: {float(last.get(\"metrics/pose_mAP50-95\", 0)):.3f}')
"
```

| 指標 | 意味 | 目標値 |
|-----|------|--------|
| pose_mAP50 | IoU50でのキーポイント検出精度 | 0.7以上 |
| pose_mAP50-95 | IoU50-95の平均 | 0.5以上 |

> 訓練データが少ない（80枚程度）場合、mAP50が0.5〜0.7でも実用可能。
> 撮影枚数を増やすと精度が上がる。

## トラブルシューティング

| 問題 | 対処 |
|-----|------|
| `No such file: dataset.yaml` | `elbow-train/dataset.yaml` の `path` が正しいか確認 |
| `CUDA out of memory` | `batch=8` に変更。さらに小さくしてもOK |
| MPS が使えない | `device='cpu'` に変更 |
| `labels not found` | ラベルファイルのパスと画像のパスが対応しているか確認 |
| pose_loss が下がらない | アノテーションのキーポイント順序が正しいか確認（0〜3の順） |
| 訓練が即終了する | データが少なすぎる（最低20枚）またはラベルファイルが空 |
