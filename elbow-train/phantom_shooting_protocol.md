# ElbowVision ファントム撮影プロトコル
## Canon 装置対応 詳細版

---

## 概要

ElbowVision の訓練データとして、以下の2種類の画像を収集する。

| 種別 | 装置 | 用途 |
|------|------|------|
| **単純X線（CR/DR）** | Canon CXDI / CXDI-NX 系 | YOLOv8-Pose 訓練データ（主力） |
| **CTボリュームデータ** | Canon Aquilion 系（1mm/0.8mm）| DRR生成・3D解析・骨格参照 |

> **ElbowVisionの強み**: 単純X線ファントム画像を直接使うためドメインギャップなし。
> CTボリュームデータは補助的にDRR生成・骨格確認に使用する。

## Part 1 : 単純X線撮影（Canon CXDI / DR）

### 装置設定

| 項目 | 設定値 | 備考 |
|------|--------|------|
| 検出器 | Canon CXDI-810C / CXDI-NX（15×15 または 17×17インチ） | |
| 収録モード | DR（ダイレクト収録） | CR（コンピューテッドラジオグラフィー）でも可 |
| 画素サイズ | 0.148 mm/pixel（CXDI-810C）/ 0.125 mm/pixel（CXDI-NX） | 機種による |
| 画像サイズ | 2244×2244（CXDI-810C）/ 2448×2448（CXDI-NX）前後 | |
| 保存形式 | **DICOM（必須）** | JPEGやPNGで保存しない |
| 処理モード | Bone Enhanced（骨強調）またはStandard | 施設の設定に合わせる |

### 撮影条件（肘ファントム）

**AP像（正面）:**

| 項目 | 設定値 | 根拠 |
|------|--------|------|
| 管電圧 | **60 kVp** | 骨相当樹脂ファントム標準値。金属代用骨は65〜68 kVp |
| 管電流時間積 | **5〜7 mAs** | S値目標: 200〜400（Canon AEC使用時はREX値400前後） |
| SID | **100 cm** | 四肢撮影標準値 |
| OID | 0〜2 cm | ファントムを検出器に密着 |
| 焦点サイズ | **小焦点（0.6 mm）** | 骨端部の鮮鋭度確保 |
| グリッド | **不使用** | 四肢撮影・体厚<10 cm |
| 照射野 | **上腕骨下1/3 〜 前腕上1/3** | 上腕骨幹・両上顆・前腕骨幹が全て収まる |
| 中心線 | **顆部中央、垂直入射** | |

**側面像:**

| 項目 | 設定値 | 備考 |
|------|--------|------|
| 管電圧 | **62 kVp** | 側面では組織重複でやや高め |
| 管電流時間積 | **6〜8 mAs** | |
| SID / OID | AP像と同じ（100 cm） | |
| その他 | AP像と同一設定 | |

### 露出調整の判断基準（Canon独自指標）

| 指標 | 目標値 | 意味 |
|------|--------|------|
| **REX値**（Relative Exposure Index） | 200〜500 | 低すぎ<200（ノイズ過多）、高すぎ>800（過露出） |
| **S値**（Canon感度値） | 150〜400 | 機種・処理設定による |
| **EI値**（Exposure Index, IEC62494準拠） | 300±100 | DI（偏差指標）が±1以内 |

> REX値は装置ごとにキャリブレーションが異なる場合あり。
> 施設の標準四肢撮影プロトコルのREX値を基準にする。

### Canon CXDI 操作上の注意

1. **処理パラメータの固定**
   - 撮影ごとに画像処理（エッジ強調・コントラスト）が変わると訓練データのばらつきが増える
   - 可能なら処理パラメータを固定した専用プロトコルをコンソールに登録する

2. **S値/REX値のCSV記録**
   - 各撮影後にコンソールに表示される値をメモ → 撮影記録シートに転記

3. **DICOM匿名化**
   - ファントムなので個人情報はないが、施設名・装置情報は残しておくと後で解析に使える

### 確定条件サマリー（撮影当日の確認用）

| 項目 | **確定値** |
|------|-----------|
| 管電圧 | **100 kVp** |
| 管電流時間積 | **50〜80 mAs** |
| スキャンモード | **Helical** |
| FOV | **180 mm**（extremity 小FOV） |
| マトリックス | **512×512** |
| **再構成スライス厚** | **1.0 mm** |
| **再構成インターバル** | **0.8 mm**（オーバーラップ） |
| **再構成カーネル** | **FC43（骨高精細）** ※ FC38〜42 でも可 |
| 軟部シリーズ | **不要**（FC43 の 1 シリーズのみで OK） |

> **コードとの対応:**
> - スライス間隔（0.8mm）は DICOM の ImagePositionPatient から自動計算される
> - HU ウィンドウ: -400〜1500（骨＋軟部をカバー、空気・金属アーチファクトは除外）
> - FC43 骨カーネルは Otsu 骨閾値自動検出の精度に直結するため必須

---

### スキャン条件（詳細）

| 項目 | 設定値 | 備考 |
|------|--------|------|
| スキャンモード | **Helical（ヘリカル）** | |
| 管電圧 | **100 kVp** | ファントムのため低線量設定可 |
| 管電流時間積 | **50〜80 mAs**（固定）| AEC使用時はSD target: 20前後 |
| Pitch Factor | **0.828〜0.938** | 機種により"ヘリカルピッチ"で設定 |
| ビームコリメーション | **0.5 mm × 80列** または **0.625 mm × 64列** | 機種により異なる |
| 回転時間 | **0.35〜0.5 s/rot** | |
| FOV（スキャン） | **150〜180 mm**（extremity用小FOV） | 肘関節部のみを含む |
| **再構成スライス厚** | **1.0 mm** | |
| **再構成インターバル** | **0.8 mm** | オーバーラップ再構成（0.8mmピッチ） |
| 再構成カーネル | **FC43（骨高精細）** | 下表参照 |
| マトリックス | **512×512** | 標準 |

> **1mm/0.8mm 設定の意味:**
> - スライス厚 1.0 mm → 前後方向の分解能確保
> - 再構成間隔 0.8 mm → 0.2 mmのオーバーラップで部分体積効果を低減
> - DRR生成時に骨端部の輪郭が滑らかになる

### 再構成カーネル（Canon Aquilion FC系）詳細

| カーネル | 特性 | ノイズ | ElbowVisionでの用途 |
|---------|------|--------|-------------------|
| FC1x系 | 軟部組織（超軟） | 最小 | 腱・軟骨確認（参考のみ） |
| FC30 | 骨・標準 | 中 | 全体形態確認。骨端部のエッジはやや甘め |
| **FC38〜FC43** | **骨・高精細** | **中高** | **DRR生成・キーポイント検出に最適** |
| FC64 | 骨・超高精細 | 多 | ノイズが目立ち過ぎ。通常不要 |

> **注意: FC番号は機種・ソフトウェアバージョンで異なる**
> FC43が存在しない機種もある。コンソールの再構成プロトコル一覧で
> 「骨」「high resolution」「sharp」と書かれたカーネルを選ぶ。
> FC38・FC40・FC41・FC42 なども同等に使用可能。

**選び方の指針:**
- 骨皮質の輪郭が鮮明に出るカーネルを選ぶ（外側上顆・内側上顆の境界が大事）
- ノイズが多すぎてザラザラになるカーネルは避ける（FC64相当は過剰）
- 迷ったら施設の「肘 単純CT」プロトコルのカーネルを流用するのが最速

### Aquilion 専用設定（コンソール操作）

```
[検査部位] 四肢（upper extremity）を選択
[体位]     頭から先（Head First）またはFeet First（プロトコルによる）
[アーム位置] ファントムを寝台中央に固定
[プロトコル] 上肢（extremity）高精細プロトコルがあれば流用可

[再構成設定]
  Series 1: 1.0mm / 0.8mm / FC43（骨高精細）← DRR生成・キーポイント検出用（主力）
  Series 2: 1.0mm / 0.8mm / FC30（骨標準）   ← 全体形態確認用
  Series 3: 3.0mm / 3.0mm / FC13（軟部組織） ← MPR確認・軟部参照用

[DFOV（表示FOV）] 180mm

※ Series 1 が ElbowVision で使用するメインシリーズ
```

### 再構成後の確認項目

```
DICOM ヘッダーで以下を確認:
  PixelSpacing:      0.351mm × 0.351mm 前後（FOV180mm / 512matrix）
  SliceThickness:    1.0
  SpacingBetweenSlices: 0.8
  ConvolutionKernel: FC43
  KVP:               100

期待される画質:
  - 外側上顆の骨皮質が2〜3ピクセル幅で描出
  - 上腕骨小頭と橈骨頭の関節隙が視認可能
  - 骨梁パターンが視認可能（ノイズに埋もれていない）
```

### Aquilion 操作上の注意

1. **ファントム固定**
   - 肘がスキャン中に動かないようにテープ・サンドバッグで固定
   - 金属固定具は使用しない（アーチファクト）

2. **スカウト画像の確認**
   - AP・側面スカウトでファントムの傾き確認
   - 上腕骨長軸がzoom軸方向になるように配置

3. **ボクセルサイズ確認**
   - 再構成後のDICOMヘッダーで `PixelSpacing` と `SliceThickness` を確認
   - DRR生成スクリプト（後述）はこれを自動読み取り

## Part 8 : FOV と画像スケールの考え方

### 単純X線（DR）の FOV：揃えなくてよい

| 項目 | 結論 | 理由 |
|-----|------|------|
| 照射野（コリメーター範囲） | **揃えなくてよい** | YOLOは座標を0〜1で正規化するため、FOVの違いは自動吸収される |
| SID（焦点-検出器間距離） | **できれば100cmに統一** | 拡大率が変わるが訓練データの多様性として許容 |
| 必須条件 | **4キーポイントが全部フレーム内に収まること** | フレームアウトしたキーポイントはアノテーション不可 |

**つまり撮影時に気にすること：** 上腕骨幹部の上端から前腕骨幹部の下端まで、
必ず1枚の画像に収まる照射野を確保すること。それだけ。

### CTボリュームの DFOV（表示FOV）：統一推奨

| 項目 | 推奨 | 理由 |
|-----|------|------|
| DFOV | **180mm に統一** | DRR生成時のピクセルサイズが揃う（mm/pixel が一定になる） |
| DFOVが違う場合 | 使用可能だが注意が必要 | DRR生成スクリプトでvoxelサイズを自動読み取るので補正は自動的に行われる |

## Part 9 : CTボリュームの肘向き問題と対処法

### 問題：ガントリー内でのファントムの向きがバラバラになりやすい

```
CT寝台方向:  ─────────────────────→ z軸（頭足方向）

パターンA：上腕骨が z軸と平行
  ┌──────────────────┐
  │  [上腕骨]→[肘]→[前腕] │   理想的な配置
  └──────────────────┘

パターンB：上腕骨が斜め（よくある）
  ┌──────────────────┐
  │    ／[上腕骨]         │
  │ ／      [肘][前腕]    │   斜め配置（DRRを作るときに補正が必要）
  └──────────────────┘

パターンC：上腕骨が横向き（Superman Position）
  ┌──────────────────┐
  │  ↑ 上腕骨              │
  │  ↓ 前腕骨              │   z軸と垂直（DRR生成には最も不向き）
  └──────────────────┘
```

### 対処方針

| 用途 | 対処 |
|-----|------|
| **単純X線の訓練データのみ使う場合** | CT向きは無関係。問題なし |
| **CTからDRRを生成したい場合** | 撮影時にファントム向きを統一する、または向き補正スクリプトを使う |

### CTでDRRを生成する場合の推奨配置

```
【推奨】上腕骨長軸を寝台長軸（z軸）と平行に配置する

正面:
          ガントリー
    ────────────────────
    ↑z軸    [上腕骨幹]──[肘関節]──[前腕骨幹]
    ────────────────────

側面: 寝台面から見て上腕骨が真っ直ぐ
```

**実際の固定方法：**
- 長めのサンドバッグでファントムを寝台長軸に沿って固定
- Scoutで確認: AP Scout で上腕骨の長軸が画面と平行になっているか確認
- 傾いていたら固定し直す（5°以内の傾きは許容）

### 向きがバラバラになってしまった場合の後処理

CTのDICOMデータから骨の主軸を自動検出して向きを揃えるスクリプトを後で作成予定。
現時点では単純X線を主力訓練データとして使うため、CT向きの影響は限定的。
