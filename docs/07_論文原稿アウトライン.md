## Abstract (Structured, 250 words max)

**Purpose:**
Accurate positioning in elbow radiography is critical for reliable diagnosis, yet positioning quality depends heavily on radiographer experience. This study proposes an automated elbow positioning assessment system that uses digitally reconstructed radiographs (DRRs) generated from CT volumes to train deep learning models, eliminating the need for manual annotation and enabling objective, real-time positioning feedback.

**Methods:**
An elbow phantom was scanned using multi-detector CT. DRRs were generated from the CT volume at various forearm rotation and elbow flexion angles using a Beer-Lambert projection algorithm (ElbowSynth), producing anteroposterior (AP) and lateral views with automatically calculated ground-truth positioning labels. A YOLOv8-pose model was trained for anatomical keypoint detection (6 keypoints including humeral shaft, medial/lateral epicondyles, forearm shaft, radial head, and olecranon), followed by a ConvNeXt regression model for quantitative positioning error estimation. Agreement between AI-estimated and CT-derived ground-truth angles was evaluated using Bland-Altman analysis, intraclass correlation coefficient (ICC), and mean absolute error (MAE).

This phantom study demonstrates the feasibility of AI-assisted radiographic positioning quality control without requiring ethical approval for patient data.

## 1. Introduction

### 1.1 背景

- 肘X線撮影におけるポジショニングの重要性
  - 不適切なポジショニングは診断精度を低下させる
  - 側面像: 内外側上顆の重なり（superimposition）が理想
  - 正面像: 適切な前腕回旋位が必要
- 現状の課題
  - ポジショニング品質は撮影者の経験に依存
  - 客観的な定量評価手法が確立されていない
  - 撮り直しによる被曝増加

### 1.2 関連研究

- AI を用いた X 線画像品質評価の先行研究
  - 胸部X線のポジショニング評価（回旋検出等）
  - 膝関節・股関節の自動角度計測
- DRR を訓練データに用いたアプローチの利点
  - 大量のラベル付きデータを自動生成可能
  - ドメインギャップの課題と対策

### 1.3 目的

- CTボリュームからDRRを自動生成し、アノテーション不要で深層学習モデルを訓練するパイプラインを構築
- ファントム実験によりAI推定角度とCT由来真値の一致性を検証
- 放射線技師の撮影技術向上を支援するリアルタイムフィードバックシステムの実現可能性を示す

## 2. Materials and Methods

### 2.1 Study Design

- ファントム検証研究（倫理委員会審査不要）
- 使用ファントム: 肘関節ファントム（[型番・メーカー]）
- **研究倫理:** 本研究はファントム（人体模型）のみを使用しており、ヒト被験者を対象としないため、施設倫理委員会の審査対象外である。

### 2.2 CT Acquisition

| パラメータ | 値 |
|-----------|-----|
| CT装置 | Canon Aquilion（Canon Medical Systems） |
| 管電圧 | 100 kVp |
| 管電流時間積 | 50--80 mAs（固定） |
| スキャンモード | Helical |
| FOV | 180 mm（extremity 小FOV） |
| マトリックス | 512 x 512 |
| 再構成スライス厚 | 1.0 mm |
| 再構成インターバル | 0.8 mm（オーバーラップ） |
| 再構成カーネル | FC43（骨高精細） |
| 撮影体位 | 肘伸展位・屈曲90度・屈曲135度 |

> PixelSpacing は FOV 180 mm / 512 matrix = 約 0.351 mm。
> スライス厚 1.0 mm / 間隔 0.8 mm のオーバーラップ再構成により、骨端部の部分体積効果を低減し DRR の輪郭品質を確保した。

### 2.3 DRR Generation (ElbowSynth)

#### 2.3.1 アルゴリズム概要

1. **DICOM読み込みと正準方向への変換**
   - ImageOrientationPatientタグからCTボリュームの向きを自動検出
   - 6方向（+/-PD, +/-AP, +/-ML）全てに対応
   - 正準方向: axis0=PD（近位→遠位）, axis1=AP（前→後）, axis2=ML（内→外）

2. **3Dアフィン変換による前腕回旋・肘屈曲**
   - SciPy `affine_transform()` を使用
   - 回旋角度: [-30, +30]度の範囲でランダムサンプリング
   - 屈曲角度: CTの基準屈曲角からの変動を付加

3. **Beer-Lambert透過投影**
   - AP方向（axis1）およびML方向（axis2）への透視投影
   - SID（X線管焦点-検出器間距離）: 1000mm
   - HUウィンドウ: [-200, 1000]（ファントム用）/ [-400, 1500]（実骨用）

4. **後処理**
   - CLAHE（Contrast Limited Adaptive Histogram Equalization）
   - リサイズ（256x256px）
   - オプション: ドメイン適応augmentation（ノイズ・ぼかし・コントラスト変動）

#### 2.3.2 自動ラベル生成

- 6キーポイント座標（YOLO形式）
  - 0: 上腕骨幹部, 1: 外側上顆, 2: 内側上顆, 3: 前腕骨幹部, 4: 橈骨頭, 5: 肘頭
- 回旋エラー角度（理想位からのズレ量）
- 屈曲角度
- 外反角（Carrying angle）

#### 2.3.3 DRR生成パイプライン技術詳細

ElbowSynth（`elbow_synth.py`）による生成パラメータ:

| パラメータ | 値 | 備考 |
|-----------|-----|------|
| `--n_ap` | 240 | AP DRR 生成枚数 |
| `--n_lat` | 720 | LAT DRR 生成枚数 |
| `--target_size` | 256 | CTボリュームのリサイズ先（px）。DRR出力も 256x256 |
| `--domain_aug` | 有効 | ガウシアンノイズ・ぼかし・コントラスト変動による実X線ドメイン適応 |
| `--sid` | 1000 mm | X線管焦点-検出器間距離 |
| HUウィンドウ | [-200, 1000]（ファントム）/ [-400, 1500]（実骨） | 組織コントラスト調整 |
| CLAHE | 有効 | 後処理コントラスト強調 |

`--domain_aug` は DRR と実X線のドメインギャップを軽減するため、各 DRR に対しガウシアンノイズ（sigma 0--10）、ガウシアンぼかし（kernel 3--7）、コントラスト変動（gamma 0.7--1.5）をランダムに適用する。

3シリーズCT（伸展180度・屈曲135度・屈曲90度）から `--series_nums 4,8,12 --base_flexions 180,135,90` で一括生成し、各シリーズ AP 240枚 + LAT 720枚 = 960枚/シリーズ を生成可能。

#### 2.3.4 生成枚数

| ビュー | 訓練 | 検証 | 合計 |
|--------|------|------|------|
| AP | 192 | 48 | 240 |
| LAT | 576 | 144 | 720 |
| **合計** | **768** | **192** | **960** |

> 実際のデータセット v2 では複数シリーズからの生成により train: 876枚 / val: 168枚 = 合計 1,044枚 が使用されている。

### 2.4 Deep Learning Models

#### 2.4.1 YOLOv8-Pose（キーポイント検出）

| パラメータ | 値 |
|-----------|-----|
| モデル | YOLOv8s-pose（small） |
| 入力サイズ | 256x256 |
| キーポイント数 | 6（各3次元: x, y, visibility） |
| エポック数 | 150（Early stopping, patience=20） |
| バッチサイズ | 64（M4 Pro 64GB RAM 活用） |
| オプティマイザ | SGD（lr0=0.01, lrf=0.01, warmup=5 epoch） |
| データ拡張 | 水平反転（flip_idx=[0,2,1,3,4,5]）, mosaic=1.0, degrees=10, scale=0.3, translate=0.1 |
| **訓練環境** | **Mac mini M4 Pro（14コアCPU, MPS GPU, 64GB RAM）** |

> M4 Pro の Apple MPS GPU（`device='mps'`）により、Google Colab（NVIDIA T4）に依存せずローカル環境のみで訓練を完結できる。960枚 / 150 epoch で約20--40分、early stopping により通常80--120 epoch で収束する。専用スクリプト `train_m4pro.py` を使用。

#### 2.4.2 ConvNeXt回帰モデル（ポジショニングエラー推定）

| パラメータ | 値 |
|-----------|-----|
| バックボーン | ConvNeXt-Small（ImageNet事前学習） |
| 出力 | rotation_error_deg, flexion_deg（2値回帰） |
| 損失関数 | Smooth L1 Loss |
| オプティマイザ | AdamW |
| 学習率 | 1e-4（cosine annealing） |
| バッチサイズ | 32（M4 Pro 64GB RAM） |
| エポック数 | 100（Early stopping, patience=20, warmup=5） |
| **訓練環境** | **Mac mini M4 Pro（MPS GPU）** |

> ConvNeXt 訓練も M4 Pro ローカルで完結する（`train_convnext_m4pro.py`）。所要時間は約10--20分。YOLO訓練と連続実行する場合は `train_m4pro.py && train_convnext_m4pro.py` でパイプライン化可能。

### 2.5 Evaluation Metrics

#### 2.5.1 キーポイント検出精度

- mAP50-pose
- PCK@10px（Percentage of Correct Keypoints）
- 各キーポイント別の誤差分布

#### 2.5.2 角度推定精度（Bland-Altman法）

- **GT（Ground Truth）:** CTボリュームから計算された3D角度（ElbowSynthが自動算出）
- **Pred:** AI推定角度
- 評価指標:
  - Mean bias（平均差）
  - 95% Limits of Agreement (LoA)
  - MAE（平均絶対誤差）
  - RMSE（二乗平均平方根誤差）
  - ICC(3,1)（級内相関係数）
  - Pearson相関係数

#### 2.5.3 臨床許容基準

| 角度 | Bias許容 | LoA許容 |
|------|---------|---------|
| Carrying angle（外反角） | +/-1度 | +/-5度 |
| Flexion（屈曲角） | +/-3度 | +/-8度 |

## 4. Discussion

### 4.1 主要な知見

- DRRベースの訓練データ自動生成により、アノテーションコストをゼロにできた
- ファントム使用により倫理的障壁なく研究を開始できた
- [結果に応じた知見]

### 4.2 DRRアプローチの利点

- **完全自動ラベル生成:** 3D→2D投影時にキーポイント座標・角度が自動計算される
- **データ量のスケーラビリティ:** パラメータ変更で任意の枚数を生成可能
- **再現性:** 同一CTから同一条件で再生成可能

### 4.3 ドメインギャップの考察

- DRR（合成画像）と実X線の画質差
- ドメイン適応augmentation（ノイズ・ぼかし・コントラスト変動）による軽減効果
- Fine-tuningによる実X線への適応可能性

### 4.4 臨床応用の展望

- リアルタイムフィードバックによる撮影技術の標準化
- 経験の浅い技師の教育支援
- 撮り直し率の低減と被曝低減

### 4.5 Limitations

- ファントム研究のため、実患者データでの検証が未実施
- 単一ファントムのため、体型・年齢による変動が未考慮
- DRRは散乱線を含まないため、実X線との物理的な差がある
- 訓練は Apple M4 Pro MPS GPU で実施しており、大規模データセットでの訓練には CUDA GPU 環境が望ましい

### 4.6 今後の研究

- 実患者CTを用いた大規模検証（IRB承認後）
- マルチセンター検証
- 他関節（膝・肩・手首）への拡張
- エッジデバイスへのモデルデプロイ

## 5. Conclusion

本研究では、CTボリュームからDRRを自動生成し、深層学習モデルを訓練する肘X線ポジショニング評価パイプラインを構築した。YOLOv8-poseによるキーポイント検出とConvNeXtによるポジショニングエラー回帰を組み合わせ、ファントム実験によりCT由来真値との一致性を[Bland-Altman解析で]検証した。本手法はアノテーション不要かつ倫理委員会審査不要で研究を開始できる利点があり、放射線技師の撮影技術支援に向けた基盤技術として有望である。
