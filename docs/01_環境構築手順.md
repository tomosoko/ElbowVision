# 01 環境構築手順

## 前提条件

| ツール | 必要バージョン | 確認コマンド |
|-------|-------------|------------|
| Python | 3.10 以上 | `python3 --version` |
| Node.js | 20 以上 | `node --version` |
| npm | 10 以上 | `npm --version` |
| Git | 任意 | `git --version` |

---

## 1. リポジトリのクローン（初回のみ）

```bash
# すでに /Users/kohei/ElbowVision_Dev/ に作成済みのため不要
cd /Users/kohei/ElbowVision_Dev
```

---

## 2. FastAPI バックエンドの環境構築

```bash
cd /Users/kohei/ElbowVision_Dev/elbow-api

# Python 仮想環境を作成
python3 -m venv venv

# 仮想環境を有効化
source venv/bin/activate
# （Windows の場合: venv\Scripts\activate）

# 依存パッケージをインストール
pip install -r requirements.txt
```

### インストール確認

```bash
python -c "from ultralytics import YOLO; print('YOLO OK')"
python -c "import fastapi; print('FastAPI OK')"
python -c "import pydicom; print('pydicom OK')"
```

### よくあるエラーと対処

| エラー | 原因 | 対処 |
|-------|------|------|
| `No module named 'cv2'` | opencv未インストール | `pip install opencv-python-headless` |
| `No module named 'ultralytics'` | ultralytics未インストール | `pip install ultralytics` |
| `torch` のインストールが遅い | 通常 | 待つ（数分かかる） |
| Apple Silicon でtorchエラー | MPS未対応バージョン | `pip install torch==2.2.2 torchvision==0.17.2` |

## 6. 動作確認（全体テスト）

### APIサーバーを起動してテスト

```bash
cd /Users/kohei/ElbowVision_Dev/elbow-api
source venv/bin/activate
python main.py
# → "ElbowVision AI API is running." が表示されれば OK
# Ctrl+C で停止
```

### ブラウザで確認

```
http://localhost:8000/api/health
```

以下のようなJSONが返れば成功:
```json
{
  "status": "ok",
  "version": "1.0.0",
  "engines": {
    "yolo_pose": false,
    "resnet_xai": false,
    "gradcam_xai": false,
    "classical_cv": true
  }
}
```

> `yolo_pose: false` は正常。モデル訓練前はこの状態。

### ユニットテストの実行

```bash
cd /Users/kohei/ElbowVision_Dev/elbow-api
source venv/bin/activate
pip install pytest httpx
pytest tests/ -v
```
