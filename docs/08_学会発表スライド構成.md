## スライド構成（口演 12枚）

### スライド 1: タイトル

**ElbowVisionの概要タイトル（例）:**
「CTボリュームからのDRR自動生成と深層学習による肘X線ポジショニング評価システム -- ファントム検証 --」

英語タイトル:
"Automated Elbow Radiograph Positioning Assessment Using DRR-based Deep Learning: A Phantom Validation Study"

- 著者名・所属
- 利益相反の開示

---

### スライド 2: 背景と課題（Background）

**内容:**
- 肘X線撮影におけるポジショニングの重要性
  - 側面像: 内外側上顆の重なりが理想
  - 不適切なポジショニング → 診断精度低下・撮り直し → 被曝増加
- 現状の課題
  - 品質は撮影者の経験に依存
  - 客観的な定量評価法が確立されていない

**使用する図表:** ポジショニング良好/不良の比較画像（実X線またはDRR）

---

### スライド 3: 研究目的（Purpose）

**内容:**
1. CTからDRRを自動生成し、アノテーション不要の訓練パイプラインを構築
2. AIによるポジショニングエラー推定精度をBland-Altman法で検証
3. リアルタイムフィードバックシステムとしての実現可能性を評価

**ポイント:**
- ファントム使用 → 倫理委員会不要 → すぐに始められる
- 完全自動ラベル生成 → 大規模データセットを低コストで構築

---

### スライド 4: システム全体像（System Pipeline）

**内容:**
CT撮影 → DRR生成（ElbowSynth）→ YOLOv8-pose訓練 → ConvNeXt訓練 → 推論

**使用する図表:** `results/figures/fig1_pipeline.png`（Figure 1）

---

### スライド 5: DRR生成アルゴリズム（Methods 1）

**内容:**
- CTボリュームの正準方向への変換（6方向自動対応）
- 3Dアフィン変換による前腕回旋・肘屈曲の変化
- Beer-Lambert透過投影（SID=1000mm）
- 後処理（CLAHE・リサイズ・augmentation）

**使用する図表:** `results/figures/fig2_drr_algorithm.png`（Figure 2）

### スライド 6: DRR生成結果（Methods 2）

**内容:**
- 生成枚数: AP 240枚 + LAT 720枚 = 合計 960枚/シリーズ（3シリーズで約1,044枚）
- 角度バリエーション: 回旋[-30, +30]度 x 屈曲変動（base_flexion: 180/135/90度）
- 6キーポイントの自動座標計算（YOLO形式）
- `--domain_aug`: ガウシアンノイズ・ぼかし・コントラスト変動で実X線らしさを付与
- `--target_size 256`: 出力画像 256x256 px
- 生成環境: Mac mini M4 Pro

**使用する図表:** `results/figures/fig3_drr_variations.png`（Figure 3）

### スライド 7: 深層学習モデル（Methods 3）

**内容:**

| | YOLOv8-Pose | ConvNeXt |
|---|---|---|
| 役割 | 6キーポイント検出 | ポジショニングエラー回帰 |
| 入力 | 256x256 DRR画像 | 256x256 DRR画像 |
| 出力 | 6点の(x,y,v)座標 | 回旋エラー角度・屈曲角度 |
| 訓練環境 | Mac mini M4 Pro（MPS GPU, 64GB RAM） | 同左 |
| 訓練時間 | 約20--40分 / 150epoch | 約10--20分 |

**補足:**
- YOLOv8はリアルタイム推論が可能（推論速度を明記）
- ConvNeXtはImageNet事前学習からFine-tuning

### スライド 8: 評価方法（Methods 4）

**内容:**
- GT（Ground Truth）: CTボリュームから算出された3D角度（ElbowSynthが自動計算）
- Bland-Altman法による一致性評価
  - 平均バイアス、95%一致限界（LoA）
  - ICC(3,1)、MAE、RMSE
- 臨床許容基準
  - 外反角: Bias +/-1度、LoA +/-5度
  - 屈曲角: Bias +/-3度、LoA +/-8度

### スライド 11: 考察（Discussion）

**内容:**
- DRRベースアプローチの利点
  - アノテーションコスト: ゼロ（完全自動）
  - データスケーラビリティ: パラメータ変更で無制限に生成可能
  - 再現性: 同一CTから再生成可能
- ドメインギャップの課題と対策
  - DRRは散乱線を含まない → augmentationで軽減
  - 実X線でのFine-tuning戦略
- 限界（Limitations）
  - 単一ファントム・単一施設
  - 実患者データでの検証が今後必要

---

### スライド 12: まとめと今後の展望（Conclusion）

**内容:**
- まとめ
  - CTからDRRを自動生成する訓練パイプラインを構築した
  - ファントム検証でCT真値との[良好な/有望な]一致性を確認した
  - アノテーション不要・倫理審査不要で即座に研究開始可能
- 今後の展望
  - 実患者CTでの検証（IRB承認後）
  - リアルタイムフィードバック機能の臨床試験
  - 他関節（膝・肩）への拡張
  - エッジデバイスへのデプロイ（撮影室内での使用）

## 質疑応答の想定Q&A

| 想定質問 | 回答ポイント |
|---------|------------|
| DRRと実X線のドメインギャップは？ | augmentationで軽減。散乱線がないのは限界だが、骨輪郭は十分再現。Fine-tuning戦略も用意 |
| なぜYOLOv8を選んだ？ | リアルタイム推論が可能。キーポイント検出と物体検出を同時にできる。医用画像での実績あり |
| 臨床応用までのロードマップは？ | ファントム（今回）→ 患者CT+実X線（IRB取得後）→ 前向き臨床研究 → 実装 |
| 他関節への拡張は？ | パイプラインは汎用的。CT+DRR生成部分を差し替えれば膝・肩にも適用可能（OsteoVision: 膝で同様のアプローチ実施済み） |
| 放射線技師がAIを使う意義は？ | 技師の判断を代替するのではなく支援する。教育ツール・品質管理ツールとしての位置付け |
| サンプルサイズは十分か？ | ファントム研究としてはProof of Concept。患者データでの大規模検証が次のステップ |
