# 02 データ準備手順（CT → DRR 自動生成 ＆ 実X線品質確認）

## 全体の流れ

ElbowVisionの訓練は **CT → DRR自動生成** で完結する。LabelStudioでの手動アノテーションは**不要**。

```
【訓練データ生成】
CT ボリューム（DICOM）
    ↓
elbow_synth.py で DRR を自動生成
（回旋: -30°〜+30°、屈曲: AP/LAT各パターン）
    ↓
YOLO ラベル & dataset_summary.csv を自動出力
    ↓
data/images/train+val/ と data/labels/train+val/ に配置済み

【DRR品質確認（任意）】
撮影した実X線（AP・LAT）
    ↓
dicom_to_png.py で PNG 変換
    ↓
LabelStudio で数枚だけキーポイント確認
（DRRと実X線の見え方が一致しているか確認するため）
```

---

## Step 1 : CT から DRR を自動生成する（メイン）

CTのDICOMを `data/raw_dicom/ct/` に配置してから実行。

```bash
cd /Users/kohei/Dev/ElbowVision_Dev
source elbow-api/venv/bin/activate

python elbow-train/elbow_synth.py \
  --ct    data/raw_dicom/ct/ \
  --out   data/ \
  --n     720
```

### オプション説明

| オプション | 説明 | 推奨 |
|----------|------|------|
| `--ct` | CTのDICOMフォルダ | |
| `--out` | 出力先（data/ 以下に images/ labels/ を自動作成） | |
| `--n` | 生成枚数（AP/LATそれぞれ各回旋角を何枚生成するか） | 720 |

### 実行後の確認

```bash
ls data/images/train/   # DRR が入っているか確認
ls data/labels/train/   # 自動ラベルが入っているか確認
cat data/yolo_dataset/dataset_summary.csv | head  # rotation_error_deg が入っているか確認
```

## Step 2 : 実X線でDRR品質を確認する（任意）

DRRが実X線と同じ見え方かを確認するため、数枚だけ手動確認する。

### 実X線をPNGに変換する

撮影したDICOMファイルを `data/raw_dicom/xray/` に配置してから実行。

```bash
cd /Users/kohei/Dev/ElbowVision_Dev
source elbow-api/venv/bin/activate

python elbow-train/dicom_to_png.py \
  --input  data/raw_dicom/xray/ \
  --output data/images/xray_check/ \
  --clahe \
  --size 512
```

### DRRと実X線を目視比較する

- DRR画像（`data/images/train/` のLAT/AP）と実X線（`data/images/xray_check/`）を並べて確認
- 上顆の位置・骨幹の傾きが一致しているか確認
- 大きくズレていれば `elbow_synth.py` のCT向きパラメータを調整

## Step 3 : LabelStudio で数枚確認する（任意・品質チェック用）

実X線でキーポイントの位置を数枚確認し、DRRの自動ラベルと見比べる。
これはDRRの品質確認であり、訓練ラベルを手動で作る作業ではない。

```bash
source /Users/kohei/Dev/ElbowVision_Dev/venv_labelstudio/bin/activate
label-studio start
```

ブラウザで `http://localhost:8080` が開く。（初回はアカウント作成が必要）

### LabelStudio プロジェクト作成（品質確認用）

1. 「Create Project」→ Project Name: `ElbowVision_QC`
2. 「Data Import」→ `data/images/xray_check/` の画像を数枚インポート
3. 「Labeling Setup」→ 「Custom template」→ 以下のXMLを貼り付け:

```xml
<View>
  <Image name="image" value="$image" zoom="true" zoomControl="true"/>
  <KeyPointLabels name="kp" toName="image" opacity="0.9" strokeWidth="3">
    <Label value="humerus_shaft"      background="#3b82f6"/>
    <Label value="lateral_epicondyle" background="#a855f7"/>
    <Label value="medial_epicondyle"  background="#ec4899"/>
    <Label value="forearm_shaft"      background="#22c55e"/>
  </KeyPointLabels>
</View>
```

4. 「Save」をクリック

## Step 4 : キーポイント位置の確認

### キーポイントの定義（必ず守る）

| ラベル | 解剖学的位置 | AP像 | 側面像 |
|--------|-----------|------|--------|
| `humerus_shaft` | 上腕骨幹部の**近位端中央** | 骨幹上端の中央 | 後方皮質の上端 |
| `lateral_epicondyle` | **外側上顆**の最外側点 | 外側の突出部 | 外側の膨らみ（外側上顆）の最外側点 |
| `medial_epicondyle` | **内側上顆**の最内側点 | 内側の突出部 | 後方に張り出した内側上顆の先端 |
| `forearm_shaft` | 前腕骨幹部の**遠位端中央** | 尺骨幹下端の中央 | 橈骨幹遠位端 |

> **重要**: AP像・側面像で**同じ解剖学的点**を一貫してマーキングすること。
> 側面像で小頭（capitulum）や肘頭（olecranon）を代替として使わないこと。
> 同一モデルがAP/LAT両ビューを学習するため、キーポイントの意味が像によって変わるとモデルが混乱する。

### AP像でのマーキング手順

```
画像をズームイン（マウスホイール）して以下の順にマーキング:

1. humerus_shaft（青）
   → 上腕骨の一番上、骨幹の中央線上

2. lateral_epicondyle（紫）
   → 関節部の外側、一番横に張り出している点

3. medial_epicondyle（ピンク）
   → 関節部の内側、一番横に張り出している点

4. forearm_shaft（緑）
   → 前腕骨の一番下、骨幹の中央線上
```

### 側面像でのマーキング手順

```
1. humerus_shaft（青）
   → 上腕骨後方皮質の上端

2. lateral_epicondyle（紫）
   → 外側上顆の最外側点（外側の膨らみ）
   ※ 小頭（capitulum）ではなく上顆そのもの

3. medial_epicondyle（ピンク）
   → 内側上顆の後方最突出点
   ※ 肘頭（olecranon）ではなく上顆そのもの

4. forearm_shaft（緑）
   → 橈骨幹の遠位端
```

### キーポイントの定義

| ラベル | 解剖学的位置 | AP像 | 側面像 |
|--------|-----------|------|--------|
| `humerus_shaft` | 上腕骨幹部の**近位端中央** | 骨幹上端の中央 | 後方皮質の上端 |
| `lateral_epicondyle` | **外側上顆**の最外側点 | 外側の突出部 | 外側の膨らみの最外側点 |
| `medial_epicondyle` | **内側上顆**の最内側点 | 内側の突出部 | 後方に張り出した内側上顆の先端 |
| `forearm_shaft` | 前腕骨幹部の**遠位端中央** | 尺骨幹下端の中央 | 橈骨幹遠位端 |

### 確認のポイント（DRRと実X線の比較）

- DRRの自動ラベルと同じ解剖点にキーポイントを配置し、ズレ量を確認
- ズレが大きい（>20px）場合は `elbow_synth.py` の投影パラメータを見直す
- ズレが小さければDRR品質OK → 訓練に進む

## トラブルシューティング

| 問題 | 対処 |
|-----|------|
| DRRと実X線の骨の見え方が大きく違う | CT向き・投影角度を `elbow_synth.py` で調整 |
| LabelStudioでキーポイントが選べない | XMLのラベリング設定を再確認。`KeyPointLabels` タグが必要 |
| DRR生成に時間がかかる | CTスライス数が多い場合は `--n` を減らして試す |
| ラベルファイルが空 | `elbow_synth.py` のエラーログを確認 |
