# 04 モデル訓練手順

## 訓練の全体像

```
CT（DICOM）
    ↓
elbow_synth.py でDRRを大量生成（自動ラベル付き）
    ↓
train_yolo_pose.py で YOLOv8-Pose 訓練
    ↓
（任意）train_angle_predictor.py で ConvNeXt 訓練
```

LabelStudioによる手動アノテーションは不要。

---

## Step 1 : DRR生成（ElbowSynth）

### 実行コマンド

```bash
cd ~/develop/Dev/vision/ElbowVision
source elbow-api/venv/bin/activate

# 単一シリーズ（簡易版）
python elbow-train/elbow_synth.py \
  --ct_dir  data/raw_dicom/ct/ \
  --out_dir data/yolo_dataset/ \
  --n_ap 120 --n_lat 360 \
  --target_size 128 \
  --domain_aug \
  --laterality R

# マルチシリーズ（本番推奨: 3ボリューム 180°/135°/90°）
python elbow-train/elbow_synth.py \
  --ct_dir "data/raw_dicom/ct_volume/ﾃｽﾄ 008_0009900008_20260310_108Y_F_000" \
  --out_dir data/yolo_dataset/ \
  --series_nums "4,8,12" \
  --base_flexions "180,135,90" \
  --n_ap 300 --n_lat 600 \
  --target_size 256 \
  --hu_min 50 --hu_max 800 \
  --domain_aug \
  --laterality R
```

### 生成されるデータ

| 種別 | 内容 | 枚数目安 |
|------|------|---------|
| AP DRR | 様々な前腕回旋角でのAP像 | 120枚 |
| LAT DRR | 理想位±30°の範囲でLAT像 | 360枚 |

各DRRに以下が自動で付与される:
- キーポイント座標（YOLO形式 .txt）
- 回旋ズレ量ラベル（理想位からの角度差）
- 屈曲角ラベル（90°からのズレ）

```
data/yolo_dataset/
├── images/train/    ← DRR画像 (PNG)
├── images/val/
├── labels/train/    ← YOLOキーポイントラベル (.txt)
├── labels/val/
└── dataset_summary.csv  ← 各画像の回旋ズレ量・屈曲角
```

## Step 2 : dataset.yaml の確認

```bash
cat ~/develop/Dev/vision/ElbowVision/elbow-train/dataset.yaml
```

```yaml
path: ~/develop/Dev/vision/ElbowVision/data/yolo_dataset
train: images/train
val:   images/val
nc: 1
names: [elbow_joint]
kpt_shape: [6, 3]
flip_idx: [0, 2, 1, 3, 4, 5]
```

キーポイント定義（6点）:
- 0: humerus_shaft（上腕骨幹部）
- 1: lateral_epicondyle（外側上顆）
- 2: medial_epicondyle（内側上顆）
- 3: forearm_shaft（前腕骨幹部）
- 4: radial_head（橈骨頭・前腕回旋の直接指標）
- 5: olecranon（肘頭・LAT屈曲角・AP/LAT判定に使用）

## Step 3 : YOLOv8-Pose 訓練

### Mac Mini M4 Pro でローカル訓練（推奨）

M4 Pro の MPS GPU を使い、Colab なしですべてローカルで訓練できる。

```bash
cd ~/develop/Dev/vision/ElbowVision
source elbow-api/venv/bin/activate

python elbow-train/train_yolo_pose.py
```

**M4 Pro MPS GPU 設定:**

`train_yolo_pose.py` 内で `device='mps'` を指定する（Apple Silicon GPU）。
MPS が使えない場合は自動的に CPU にフォールバックする。

```python
# train_yolo_pose.py の訓練設定例
model.train(
    data='elbow-train/dataset.yaml',
    epochs=150,
    imgsz=256,
    batch=16,        # M4 Pro なら 16〜32 で安定
    device='mps',    # Apple Silicon MPS GPU
    patience=30,     # early stopping
)
```

**所要時間の目安（M4 Pro）:**
- 960枚 / 150 epoch: 約20〜40分
- early stopping で通常 80〜120 epoch で収束

> **参考:** Google Colab で訓練する場合は `elbow-train/elbow_colab_train.ipynb` を使用できるが、
> M4 Pro があれば Colab は不要。

### 訓練ログの見方

```
Epoch  1/100  ...  box_loss: 2.341  pose_loss: 3.812  ← 最初は高い
Epoch 20/100  ...  box_loss: 0.823  pose_loss: 1.241  ← 下がれば正常
Epoch 50/100  ...  box_loss: 0.412  pose_loss: 0.634
```

### 精度確認

| 指標 | 目標値 |
|------|--------|
| pose_mAP50 | 0.7以上 |
| pose_mAP50-95 | 0.5以上 |

## Step 4 : ConvNeXt 訓練（任意）

YOLOv8だけでも動くが、ConvNeXtを加えるとポジショニングズレ量の回帰精度が上がる。

### Mac Mini M4 Pro でローカル訓練（推奨）

```bash
cd ~/develop/Dev/vision/ElbowVision/elbow-api
source venv/bin/activate

# convnext_labels.csv を使用（dataset_summary.csv でも可）
python training/train_angle_predictor.py \
  --csv  ../data/yolo_dataset/convnext_labels.csv \
  --imgs ../data/yolo_dataset/
```

**M4 Pro での注意点:**
- PyTorch MPS バックエンドを自動検出して使用する
- batch_size=32 程度で安定（メモリ不足時は 16 に下げる）
- 所要時間: 約10〜20分（M4 Pro MPS GPU）

**出力: [rotation_error_deg, flexion_deg]（2出力）**
- AP像: rotation_error_deg（理想回外位からのズレ°）のみ損失計算
- LAT像: flexion_deg（目標90°からのズレ）のみ損失計算

> **参考:** Google Colab での訓練は `elbow_colab_train.ipynb` のセル 10〜11 でも可能だが、
> M4 Pro があればローカルで十分。

保存先: `elbow-api/elbow_convnext_best.pth`（APIが自動で読み込む）

## Step 6 : 実X線でDRR品質確認

同じファントム（または同一患者）の実X線と生成したDRRを目視で比較する。

確認ポイント:
- 骨の輪郭がDRRと実X線で一致しているか
- 内外側上顆の見え方がDRRで再現されているか
- ドメインギャップが許容範囲か

差異が大きい場合 → `elbow_synth.py` のコントラスト・シャープネス設定を調整する。

## トラブルシューティング

| 問題 | 対処 |
|-----|------|
| DRR生成が遅い | `--n_lat` を減らす（最低100枚）。M4 Pro なら本番設定でも10〜20分 |
| `No DICOM files found` | `data/raw_dicom/ct/` にDICOMが入っているか確認 |
| pose_loss が下がらない | キーポイント順序を確認（0:上腕骨幹, 1:外側上顆, 2:内側上顆, 3:前腕骨幹, 4:橈骨頭, 5:肘頭） |
| pose_mAP50 ≈ 0 | ラベルのフィールド数を確認（23=6kp正常、17=4kp旧形式）。dataset.yamlのkpt_shape: [6,3]と一致しているか |
| ファントムのランドマークがズレる | 骨閾値が低すぎる可能性。2段階Otsuが動作しているか確認（ファントムは骨素材で外殻が均一） |
| DRRと実X線の差が大きい | hu_min=50, hu_max=800（ファントム最適値）を使用。`--domain_aug`を有効に |
