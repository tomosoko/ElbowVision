# ElbowVision 全体ワークフロー

## プロジェクト概要

肘X線撮影における**最適ポジショニングをAIがリアルタイムでガイドするシステム**。

- 正面X線を見て → 側面撮影時のポジショニング注意点を提示
- 側面X線を見て → ズレ量と修正方向を提示

**訓練データ:** CT → DRR（様々な回旋角度で自動生成）→ 自動ラベル付け → アノテーション不要

**精度検証:** CT由来の真値角度 vs AI推定（ゴールドスタンダード比較）

**前研究:** ファントムCT + ファントムX線
**本研究:** 患者CT + 患者X線（臨床画像） ← パイプラインは同じ

---

## 全体の流れ

```
Step 1  環境構築
        │  Python venv / npm install
        ▼
Step 2  CT撮影（職場）
        │  肘ファントム（前研究）または患者（本研究）のCT撮影
        │  → data/raw_dicom/ct/ に DICOM を配置
        ▼
Step 3  DRR生成 + 自動ラベル付け（アノテーション不要）
        │  python elbow-train/elbow_synth.py
        │  CTボリュームから様々な前腕回旋角度でDRRを大量生成:
        │    - AP像: 前腕回外位を基準に ±回旋バリエーション
        │    - LAT像: 内外側上顆が重なる理想位を基準に ±回旋バリエーション
        │  各DRRに「理想位からの回旋ズレ量」ラベルが自動でつく
        │  3D→2D投影でキーポイント座標も自動計算
        │  → data/yolo_dataset/ に画像・ラベル・dataset.yaml が揃う
        ▼
Step 4  YOLOv8-Pose 訓練
        │  python elbow-train/train_yolo_pose.py
        │  → elbow-api/best.pt
        ▼
Step 5  ConvNeXt ポジショニング回帰訓練（任意・精度向上用）
        │  python elbow-api/training/train_angle_predictor.py
        │  → elbow-api/elbow_convnext_best.pth
        ▼
Step 6  実X線でDRR品質確認
        │  同じファントム（または同一患者）の実X線を撮影
        │  → DRRと実X線を比較してドメインギャップを確認
        │  → ドメインギャップが小さければ訓練データとして有効
        ▼
Step 7  システム起動・動作確認
        │  API: http://localhost:8000
        │  UI:  http://localhost:3000
        ▼
Step 8  精度検証（Bland-Altman）
        │  CT由来の真値ポジショニング角度 vs AI推定
        │  → 実X線に対してAIがどれだけ正確にズレを検出できるか
```

## 側面撮影の最適ポジショニング基準

| 基準 | 詳細 |
|------|------|
| 肘屈曲 | 90° |
| 内外側上顆 | 完全に重なる（superimposed） |
| 橈骨頭・尺骨 | 真横から見える |

**AIが教えること:**
- 内外側上顆がどれだけズレているか（ピクセル・角度）
- どちらの方向に何度回旋させれば理想位になるか

---

## DRR生成の仕組み（ElbowSynth）

```
CTボリューム（3D）
    │
    ├── 前腕を X° 回旋させた状態でDRR生成
    │       ↓
    │   内外側上顆の見え方が変わる
    │       ↓
    │   X=0°  → 理想的なLAT（上顆が重なる）
    │   X=15° → 軽度回旋不足
    │   X=30° → 中等度回旋不足
    │   X=-15°→ 過回旋
    │
    └── 各DRRに自動ラベル: {rotation_error: X°, keypoints: [...]}
```
