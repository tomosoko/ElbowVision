# 02 アノテーション手順（DICOM変換 → LabelStudio）

## 全体の流れ

```
撮影済みDICOM
    ↓
dicom_to_png.py で PNG 変換（CLAHE強調・512px・train/val分割）
    ↓
LabelStudio にインポート
    ↓
4点キーポイントをマーキング
    ↓
YOLO形式でエクスポート
    ↓
data/labels/train/ と data/labels/val/ に配置
```

---

## Step 1 : DICOM を PNG に変換する

撮影したDICOMファイルを `data/raw_dicom/xray/` に配置してから実行。

```bash
cd /Users/kohei/ElbowVision_Dev
source elbow-api/venv/bin/activate

python elbow-train/dicom_to_png.py \
  --input  data/raw_dicom/xray/ \
  --output data/images/all/ \
  --clahe \
  --size 512 \
  --split
```

### オプション説明

| オプション | 説明 | 推奨 |
|----------|------|------|
| `--clahe` | CLAHEコントラスト強調（骨の視認性UP） | ON |
| `--size 512` | 512×512にリサイズ | 推奨（訓練と同じサイズ） |
| `--split` | train:val = 80:20 で自動分割 | ON |
| `--no-clahe` | CLAHEをOFFにする | 比較用 |

### 実行後の確認

```bash
ls data/images/train/   # 訓練用PNG が入っているか確認
ls data/images/val/     # 検証用PNG が入っているか確認
```

## Step 3 : LabelStudio プロジェクト作成

1. 「Create Project」をクリック
2. Project Name: `ElbowVision`
3. 「Data Import」タブ → `data/images/train/` と `data/images/val/` の画像をドロップ
4. 「Labeling Setup」タブ → 「Custom template」を選択

### ラベリング設定（XMLをそのまま貼り付ける）

```xml
<View>
  <Image name="image" value="$image" zoom="true" zoomControl="true"/>
  <KeyPointLabels name="kp" toName="image" opacity="0.9" strokeWidth="3">
    <Label value="humerus_shaft"      background="#3b82f6"/>
    <Label value="lateral_epicondyle" background="#a855f7"/>
    <Label value="medial_epicondyle"  background="#ec4899"/>
    <Label value="forearm_shaft"      background="#22c55e"/>
  </KeyPointLabels>
</View>
```

5. 「Save」をクリック

## Step 4 : アノテーション作業

### キーポイントの定義（必ず守る）

| ラベル | 解剖学的位置 | AP像 | 側面像 |
|--------|-----------|------|--------|
| `humerus_shaft` | 上腕骨幹部の**近位端中央** | 骨幹上端の中央 | 後方皮質の上端 |
| `lateral_epicondyle` | **外側上顆**の最外側点 | 外側の突出部 | 外側の膨らみ（外側上顆）の最外側点 |
| `medial_epicondyle` | **内側上顆**の最内側点 | 内側の突出部 | 後方に張り出した内側上顆の先端 |
| `forearm_shaft` | 前腕骨幹部の**遠位端中央** | 尺骨幹下端の中央 | 橈骨幹遠位端 |

> **重要**: AP像・側面像で**同じ解剖学的点**を一貫してマーキングすること。
> 側面像で小頭（capitulum）や肘頭（olecranon）を代替として使わないこと。
> 同一モデルがAP/LAT両ビューを学習するため、キーポイントの意味が像によって変わるとモデルが混乱する。

### AP像でのマーキング手順

```
画像をズームイン（マウスホイール）して以下の順にマーキング:

1. humerus_shaft（青）
   → 上腕骨の一番上、骨幹の中央線上

2. lateral_epicondyle（紫）
   → 関節部の外側、一番横に張り出している点

3. medial_epicondyle（ピンク）
   → 関節部の内側、一番横に張り出している点

4. forearm_shaft（緑）
   → 前腕骨の一番下、骨幹の中央線上
```

### 側面像でのマーキング手順

```
1. humerus_shaft（青）
   → 上腕骨後方皮質の上端

2. lateral_epicondyle（紫）
   → 外側上顆の最外側点（外側の膨らみ）
   ※ 小頭（capitulum）ではなく上顆そのもの

3. medial_epicondyle（ピンク）
   → 内側上顆の後方最突出点
   ※ 肘頭（olecranon）ではなく上顆そのもの

4. forearm_shaft（緑）
   → 橈骨幹の遠位端
```

### アノテーションの品質基準

- 各キーポイントは骨の**中心または最突出点**に配置
- ズームして配置する（1ピクセル単位の精度は不要、±5pxで可）
- 視認できない点は **スキップしない**（すべてマーキング必須）
- 迷った場合は `annotation_guide.md` の図解を参照

## Step 5 : YOLO形式でエクスポート

1. プロジェクトの「Export」をクリック
2. 形式: **YOLO with Keypoints** を選択
3. ダウンロードされたZIPを解凍
4. 中身を `data/labels/` に配置:

```bash
# エクスポートしたラベルファイルを配置
cp -r export/labels/train/* data/labels/train/
cp -r export/labels/val/*   data/labels/val/
```

### ラベルファイルの確認

```bash
# ファイルが存在するか確認
ls data/labels/train/ | head -5

# 中身の確認（1行 = 1物体 = 1肘関節）
cat data/labels/train/AP_B_001.txt
# → 0 0.5 0.5 0.8 0.9 0.21 0.12 2 0.45 0.48 2 0.55 0.48 2 0.50 0.85 2
# 　 クラス Bboxcx cy w h kp0x y vis kp1x y vis kp2x y vis kp3x y vis
```

## トラブルシューティング

| 問題 | 対処 |
|-----|------|
| LabelStudioでキーポイントが選べない | XMLのラベリング設定を再確認。`KeyPointLabels` タグが必要 |
| エクスポートにYOLO形式がない | LabelStudio 1.7以上が必要。`pip install label-studio --upgrade` |
| ラベルファイルの行数が0 | アノテーションが保存されていない。Submitボタンを押したか確認 |
| 画像が表示されない | ファイルパスを確認。絶対パスでインポートするか、LabelStudioのMedia設定を確認 |
